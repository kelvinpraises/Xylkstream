Project Summary

Three problems. One protocol.

Xylkstream is a privacy-first streaming payment protocol where every dollar earns yield in transit, managed by an AI agent, controlled by you.

01

Streaming Payments

Continuous per-second money flows. Allowances, payroll, group treasuries — programmable as a single on-chain primitive.

02

ZK Privacy

Stealth wallets + E2E encrypted circles + ERC-8065 ZWERC20. Privacy is the substrate — not a toggle.

03

AI Yield Agent

Autonomous agent writes Solidity strategies, sandboxes them, proposes deployment. Agent proposes. Human approves. Browser signs.

How It Works

End-to-end: from social login to yield.

Step 1

Login → Stealth Wallet

Privy social login. One signMessage derives a 32-byte seed. WDK creates an ERC-4337 Safe with zero on-chain history. Paymaster sponsors all gas.

Step 2

Create a Circle

Sender generates an x25519 keypair. Invite link carries the public key. Members encrypt their stealth address for the sender only. Server stores ciphertext it cannot read.

Step 3

Deposit → ZK Shield

USDC deposited → minted as zwUSDC at a privacy address. Groth16 proof (2.6s in browser) remints to stream recipients. No on-chain link between sender and stream.

Step 4

AI Manages Yield

Agent connects via MCP. Writes Solidity strategies. Compiled in-process by solc-js, tested in a v8 isolate. Proposes deployment. You approve once.

Privacy overhead: ~$0.46 per stream — one-time cost. Ongoing streaming adds zero overhead.

Future Roadmap

What we learned. Where we go next.

Key learnings from the build — and the features that matter most if we keep going.

Next Up

Shareable Claims Page

The claims link should go to the recipient, not the sender. A shareable URL the streamer sends so anyone can claim without navigating the dashboard.

Next Up

Stream Notifications

Email / push / on-chain event listener. Recipients should know when money starts flowing to them — without polling the chain themselves.

Expansion

Cross-Chain Streaming

Bridge integration or multi-chain Drips deployments. Stream from BSC to Arbitrum. Let yield strategies reach deeper liquidity on any EVM chain.

Learning

Separate Paymaster Keys

Paymaster signer and bundler executor share the same key today. Production needs a dedicated PAYMASTER_SIGNER_KEY — zero shared trust between roles.

Expansion

More Yield Strategies

Venus Protocol lending, Alpaca leveraged farming, and an open plugin system so the community can submit strategies. Agent gets smarter as the menu grows.

Polish

Error UX Hardening

Apply friendlyTxError() across remaining ~9 files. Replace raw toast errors with human-readable feedback everywhere in the app.

Biggest learning: privacy and yield don't need to trade off. The ZK layer adds ~$0.46 once. The yield it enables compounds forever.
